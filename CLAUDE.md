# 時限ポップアップ Chrome拡張機能

## プロジェクト概要
学校サイト上の「n限」「n限目」表記にホバーすると、対応する時間帯をツールチップで表示するChrome拡張機能。

## チーム構成と実行スコープ

### Tech Lead（チームリード）
- チーム全体の進行管理・調整
- エージェント間の連携・メッセージルーティング
- 許可が必要なコマンド（Bash等）の代理実行
- 最終的な統合レビュー

### PdM（プロダクトマネージャー）
- 機能要件・仕様の策定（`docs/spec.md`）
- ユーザーストーリーの定義
- 対象テキストパターンの定義（半角・全角・漢数字対応）
- デフォルト設定値の決定
- 技術制約・前提条件の整理（Manifest V3等）
- 仕様の整合性チェック・チームへの共有

### デザイナー（UI/UX）
- UI/UXデザインガイドラインの策定（`docs/design.md`）
- ツールチップのビジュアル仕様（位置・サイズ・色・アニメーション）
- ポップアップ画面のデザイン（ON/OFF切替、対象サイト表示）
- オプション（設定）画面のデザイン（URL管理、時限設定）
- CSSコード・色パレットの設計

### プログラマ
- Chrome拡張機能の実装全般
- `manifest.json` — Manifest V3設定
- `content.js` / `content.css` — テキスト検出（TreeWalker）＆ツールチップ表示
- `popup.html` / `popup.js` / `popup.css` — ポップアップ画面
- `options.html` / `options.js` / `options.css` — 設定画面
- `background.js` — サービスワーカー
- `icons/` — アイコン生成（Tech Leadが代理実行）

## 運用ルール
- プログラマがBash等の許可が必要なコマンドを実行する際は、Tech Leadが代理実行する
- 仕様変更はPdMが `docs/spec.md` を更新し、チームに共有する
- デザイン変更はデザイナーが `docs/design.md` を更新し、プログラマに共有する
