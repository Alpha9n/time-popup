# 時限ポップアップ Chrome拡張機能 — 機能仕様書

## 1. 概要

学校・大学のWebサイトに表示される「1限」「2限目」などの時限表記にマウスカーソルを合わせると、対応する開始・終了時刻をツールチップで表示するChrome拡張機能。

ユーザーが指定したサイトでのみ動作し、時限ごとの時刻はユーザーが自由にカスタマイズできる。

---

## 2. ユーザーストーリー

| # | ストーリー |
|---|-----------|
| US-1 | 学生として、大学サイト上の「1限」にカーソルを合わせると「9:00〜10:30」とツールチップが出るので、別ページで時間割を確認する手間が省ける。 |
| US-2 | 学生として、拡張機能の設定画面から各時限の開始・終了時刻を変更できるので、自分の大学の時間割に合わせられる。 |
| US-3 | 学生として、拡張機能を有効にするWebサイトのURLを指定できるので、関係ないサイトでは動作しない。 |
| US-4 | 学生として、全角数字の「２限目」や半角の「2限」のどちらにもツールチップが表示されるので、表記揺れを気にせず使える。 |
| US-5 | 学生として、ポップアップ画面から拡張機能の有効/無効を素早く切り替えられる。 |

## 3. 機能一覧

### 3.1 対象テキストの検出（Content Script）

- ページ読み込み完了後、DOM内のテキストノードを走査する。
- 後述の正規表現パターンに一致する文字列を `<span>` 要素でラップし、ツールチップ表示用の属性を付与する。
- 動的に追加されるDOMにも対応するため、`MutationObserver` で監視する。

### 3.2 ツールチップ表示

- ラップした `<span>` 要素に `mouseenter` / `mouseleave` イベントを設定する。
- カーソルホバー時に、対応する時限の「開始時刻〜終了時刻」をツールチップとして表示する。
- ツールチップはカーソル付近にフローティング表示し、画面端では位置を自動調整する。
- ツールチップのスタイルはページのCSSに干渉しないよう Shadow DOM またはインラインスタイルで実装する。

### 3.3 ポップアップ画面（Popup）

- 拡張機能アイコンクリックで表示される小さなパネル。
- 機能の有効/無効トグル。
- 現在のサイトが対象サイトかどうかの表示。
- 「設定を開く」ボタンでオプション画面へ遷移。

### 3.4 オプション画面（Options Page）

- 対象サイトURL一覧の管理（追加・編集・削除）。
- 時限数の設定（最大何限まで設定するか）。
- 各時限の開始時刻・終了時刻の設定。
- 設定のインポート/エクスポート（JSON形式）。
- デフォルト設定に戻すボタン。

### 3.5 設定の保存

- `chrome.storage.sync` を使用し、Googleアカウント間で設定を同期する。
- ストレージ容量上限（sync: 100KB）に留意する。

## 4. 対象テキストパターン

以下の正規表現で検出する:

```
/([1-9１-９][0-9０-９]?)\s*時?限(目)?/g
```

### マッチ例

| テキスト | マッチ | 抽出される限数 |
|---------|--------|--------------|
| 1限     | ○     | 1            |
| 1限目   | ○     | 1            |
| ２限    | ○     | 2            |
| ２限目  | ○     | 2            |
| 10限    | ○     | 10           |
| １０限目 | ○    | 10           |
| 1時限   | ○     | 1            |
| ２時限目 | ○    | 2            |
| 0限     | ✕     | —            |
| 限目    | ✕     | —            |

- 全角数字は半角に正規化してから時限番号として扱う。
- ユーザーが設定した最大限数を超える数字はマッチしてもツールチップを表示しない。

## 6. 設定項目一覧

| 項目 | キー | 型 | デフォルト値 | 説明 |
|------|------|----|-------------|------|
| 有効/無効 | `enabled` | boolean | `true` | 拡張機能全体の有効/無効 |
| 対象サイトURL | `targetSites` | string[] | `[]` | ワイルドカード対応のURLパターン |
| 時限数 | `periodCount` | number | `7` | 設定する時限の数（1〜15） |
| 各時限の時刻 | `periods` | object[] | 下記参照 | 各時限の開始・終了時刻 |

### デフォルト時限設定（一般的な大学の時間割）

| 時限 | 開始 | 終了 |
|------|------|------|
| 1限  | 09:00 | 10:30 |
| 2限  | 10:40 | 12:10 |
| 3限  | 13:00 | 14:30 |
| 4限  | 14:40 | 16:10 |
| 5限  | 16:20 | 17:50 |
| 6限  | 18:00 | 19:30 |
| 7限  | 19:40 | 21:10 |

### periods の構造

```json
{
  "periods": [
    { "period": 1, "start": "09:00", "end": "10:30" },
    { "period": 2, "start": "10:40", "end": "12:10" },
    ...
  ]
}
```

## 7. 技術制約・前提条件

### 7.1 Manifest V3

- Chrome拡張は **Manifest V3** で実装する。
- Service Worker（バックグラウンドスクリプト）は永続化できないため、状態管理は `chrome.storage` に依存する。
- `content_scripts` はユーザーが指定したサイトURLに基づき動的に登録する（`chrome.scripting.registerContentScripts`）。

### 7.2 権限

必要な権限:

| 権限 | 用途 |
|------|------|
| `storage` | 設定の保存・同期 |
| `activeTab` | 現在のタブ情報の取得 |
| `scripting` | Content Scriptの動的登録 |
| `host_permissions`（動的） | ユーザー指定サイトへのアクセス |

### 7.3 ブラウザ互換性

- **対象ブラウザ**: Google Chrome（最新安定版）
- Chromiumベースブラウザ（Edge等）でも動作する可能性があるが、公式サポート対象外。

### 7.4 パフォーマンス

- DOM走査はページ読み込み完了後に1回実行し、以降は `MutationObserver` で差分のみ処理する。
- 大量のテキストノードがあるページでも体感速度に影響しないよう、処理を分割（requestIdleCallback等）する。
- ツールチップ要素はホバー時に動的生成し、ホバー解除時に削除する。

### 7.5 制約事項

- `<iframe>` 内のコンテンツには対応しない（V1スコープ外）。
- PDFビューア内のテキストには対応しない。
- `<input>`, `<textarea>` 内のテキストは検出対象外とする。
- Content Security Policy（CSP）が厳格なサイトでは動作しない場合がある。
